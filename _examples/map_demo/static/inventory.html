Limited Inventory Shop

Demonstrates Compare-And-Swap (CAS) for atomic inventory updates. Multiple users can buy simultaneously without overselling!

How it works: Each purchase uses CAS (Compare-And-Swap) - reads current stock, decrements it, and writes back only if no one else modified it. Conflicts auto-retry. Backend adds artificial 2 sec delay to test out concurrent operations from different tabs. The snapshot stores full item state, while the stream sends incremental transactions that clients apply as deltas.
